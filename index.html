Git Staging

The Three States

Your files can have three states:

  1. Modified
  2. Staged
  3. Commited

1. Working Directory

This is the files you have in your computer, the ones you are currently working.
When you edit the files they are modified.

2. The Staging Area

After you've done your changes, you can add files to the staging area with: 

$ git add <filename> (Add files)
$ git add --all (To add all files)
An example: 
$ git add git.c git.h git.o
These files you added to the staging area will be commited to your local database.

3. Commiting

When you have added all the files you wish to commit just run: 

$ git commit -m "Your commit message"
Some examples: 
$ git commit -m "Add slide about pushing to a repository"
After commiting your files, the data will be safely stored on your local database.

Branches

Why Branches?

  • Separate in development code from code that is already working
  • Separate Features in development from the rest of the code
  • Fix bugs

Branching

To create a branch 

$ git branch test
To change branch 
$ git checkout test

Or better yet! 
$ git checkout -b test

MERGING

Merging

Merging is when you push your finished code into another branch.
Merge your code when you have finished working on your feature, bug fix, etc.

Merging test branch into master

  1. Change to the branch you wish to merge your work
    2. $ git checkout master
  2. Merge the changes
    3. $ git merge test

All commits done? Want to share what you've done?
This is when to use push, so the changes you made on your computer propragate to the repository.
